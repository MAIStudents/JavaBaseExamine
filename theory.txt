1. Основные методы для работы с коллекциями. Классы Collections, Arrays.

copy(coll1, coll2) - копирует все элементы из col1 в col2

sort(coll) - сортирует в порядке возрастания

min(coll), max(coll) - нахождение максимума и минимума

reverse(coll) - переворачивает коллекцию



Класс Arrays предоставляет методы для работы с массивами

Примеры: где numbers - это  int[] numbers

Arrays.sort(numbers);
Arrays.copyOf(numbers, numbers.length); 

Класс Collections предоставляет методы для работы с коллекциями

Примеры: где coll - коллекция
Collections.sort(coll);
Collections.reverse(coll);

2. Коллекции Java. Иерархия классов и интерфейсов для коллекций. Примеры.

Java Collection Framework - это собрание классов и интерфейсов в Java, 
предназначенных для хранения и обработки данных. 
Он предоставляет различные абстракции для работы с группами объектов.

Несколько основных интерфейсов: Collection, List, Set, Queue, Deque и Map.

Корневым интерфейсом для коллекций, у которых есть возможность перебирать элементы, является Iterable.
От Iterable наследуется интерфейс Collection.
Он включает в себя базовые методы, такие как add(), remove(), contains(), size() и другие.

Collection имеет три основных подинтерфейса: List, Set и Queue.

List представляет собой упорядоченную коллекцию, которая может содержать дубликаты. Примерами: ArrayList, LinkedList.
Set – это коллекция, которая не может содержать дубликаты. Примеры: HashSet, LinkedHashSet, TreeSet.
Queue используется для хранения элементов, которые обрабатываются в определенном порядке. Примеры: PriorityQueue, LinkedList

Интерфейс Queue расширяется при помощи интерфейса Deque. Он позволяет работать с двумя сторонами очереди.

Также существует отдельный интерфейс, не зависящий от Collection - Map. Примеры: HashMap, TreeMap, LinkedHashMap.

Он работает по принципу пар (ключ-значение) и не может содержать несколько одинаковых ключей.
