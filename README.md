# Теория

## Вопрос 1
Класс `ArrayDeque` в Java представляет собой реализацию двусторонней очереди (дека), основанную на динамическом массиве. Эта структура данных обеспечивает эффективные операции вставки и удаления как с начала, так и с конца деки. Давайте рассмотрим основные методы и примеры использования:

### Основные методы класса `ArrayDeque`:

1. **Добавление элемента в конец деки:**
   - Методы: `addLast(element)`, `offerLast(element)`

   ```java
   ArrayDeque<String> deque = new ArrayDeque<>();
   deque.addLast("One");
   deque.offerLast("Two");
   deque.offerLast("Three");
   ```

2. **Добавление элемента в начало деки:**
   - Методы: `addFirst(element)`, `offerFirst(element)`

   ```java
   ArrayDeque<String> deque = new ArrayDeque<>();
   deque.addFirst("Zero");
   deque.offerFirst("MinusOne");
   ```

3. **Получение и удаление элемента с начала деки:**
   - Методы: `removeFirst()`, `pollFirst()`

   ```java
   ArrayDeque<String> deque = new ArrayDeque<>();
   deque.offerLast("One");
   deque.offerLast("Two");
   deque.offerLast("Three");

   String firstElement = deque.pollFirst();
   ```

4. **Получение и удаление элемента с конца деки:**
   - Методы: `removeLast()`, `pollLast()`

   ```java
   ArrayDeque<String> deque = new ArrayDeque<>();
   deque.offerLast("One");
   deque.offerLast("Two");
   deque.offerLast("Three");

   String lastElement = deque.pollLast();
   ```

5. **Получение, но не удаление, элемента с начала или конца деки:**
   - Методы: `getFirst()`, `peekFirst()` для начала, `getLast()`, `peekLast()` для конца

   ```java
   ArrayDeque<String> deque = new ArrayDeque<>();
   deque.offerLast("One");
   deque.offerLast("Two");
   deque.offerLast("Three");

   String firstElement = deque.peekFirst();
   ```

### Пример использования `ArrayDeque`:

```java
import java.util.ArrayDeque;

public class ArrayDequeExample {
    public static void main(String[] args) {
        ArrayDeque<String> deque = new ArrayDeque<>();

        // Добавление элементов в конец деки
        deque.offerLast("One");
        deque.offerLast("Two");
        deque.offerLast("Three");

        // Добавление элемента в начало деки
        deque.offerFirst("Zero");

        // Получение и удаление элемента с начала деки
        String firstElement = deque.pollFirst();
        System.out.println("First Element: " + firstElement);

        // Получение и удаление элемента с конца деки
        String lastElement = deque.pollLast();
        System.out.println("Last Element: " + lastElement);

        // Вывод содержимого деки
        System.out.println("Deque: " + deque);
    }
}
```

В данном примере элемент "One" будет удален с начала деки, элемент "Three" - с конца, и в результате получится дека "Two, Zero". Операции вставки и удаления в `ArrayDeque` обычно выполняются за константное время (O(1)), что делает его эффективной структурой данных для многих сценариев использования.
## Вопрос 2
Исключения в Java - это объекты, которые представляют собой аномальные или ошибочные ситуации, возникающие во время выполнения программы. Исключения в Java делятся на два основных типа: проверяемые (checked) и непроверяемые (unchecked).

### Проверяемые исключения (Checked Exceptions):

1. **Проверяемые исключения** - это исключения, которые должны быть обработаны в коде программы, иначе компилятор не позволит вам скомпилировать программу. Это могут быть исключения, которые могут возникнуть в результате внешних факторов, таких как файл не найден, сетевое соединение потеряно и т.д.

   Примеры проверяемых исключений:

   - `IOException`
   - `FileNotFoundException`
   - `SQLException`

   Обработка проверяемых исключений может быть выполнена с использованием блока `try-catch` или объявления метода с помощью `throws`.

   ```java
   try {
       // Код, который может вызвать проверяемое исключение
   } catch (IOException e) {
       // Обработка исключения
   }
   ```

### Непроверяемые исключения (Unchecked Exceptions):

2. **Непроверяемые исключения** (или рантайм-исключения) - это исключения, которые могут возникнуть в процессе выполнения программы, но их обработка не обязательна. Обычно они связаны с ошибками программирования или некорректными данными.

   Примеры непроверяемых исключений:

   - `NullPointerException`
   - `ArrayIndexOutOfBoundsException`
   - `ArithmeticException`

   Непроверяемые исключения обычно не требуют явного обработчика и могут быть обработаны, но это не обязательно.

   ```java
   try {
       // Код, который может вызвать непроверяемое исключение
   } catch (NullPointerException e) {
       // Обработка исключения (опционально)
   }
   ```

### Ключевые слова при использовании исключений:

1. **try:** Определяет блок кода, в котором может возникнуть исключение.

2. **catch:** Определяет блок кода, который обрабатывает исключение. Может быть несколько блоков `catch` для различных типов исключений.

3. **finally:** Определяет блок кода, который будет выполнен всегда, независимо от того, возникло исключение или нет.

4. **throw:** Используется для явного выбрасывания исключения в коде.

5. **throws:** Используется в сигнатуре метода для указания, что метод может бросить определенные типы исключений, и оставляет обработку исключений вызывающему коду.

6. **try-with-resources:** Вводится в Java 7 для управления ресурсами (например, файлы или сетевые соединения) в блоках `try-catch`, автоматически закрывая их после завершения блока.

Пример использования `try-catch`:

```java
try {
    // Код, который может вызвать исключение
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Обработка исключения
    System.out.println("Деление на ноль!");
} finally {
    // Блок finally выполняется всегда
    System.out.println("Блок finally");
}
```

Пример использования `throws` в сигнатуре метода:

```java
public void readFile(String fileName) throws IOException {
    // Код, который может вызвать исключение
    FileReader fileReader = new FileReader(fileName);
    // ...
}
```
